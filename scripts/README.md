# Скрипты

## Шапка и подвал (один экземпляр на весь сайт)

Шапка и подвал хранятся в **одних файлах** и подставляются на все страницы через JS.

- **Фрагменты (редактировать здесь — меняется на всех страницах):**
  - `assets/html/header.html` — навигация (логотип, меню, кнопка «Связаться»)
  - `assets/html/footer.html` — подвал (логотип, ссылки, контакты, копирайт)
- **Подстановка:** скрипт `assets/js/header-footer.js` вставляет шапку и подвал сразу (HTML встроен в скрипт, без запросов к серверу). Ссылки и пути к картинкам делаются относительными от текущей страницы, чтобы всё работало с `file://` и из подпапок.
- **После изменения** `header.html` или `footer.html` нужно пересобрать скрипт:
  ```bash
  python3 scripts/build_header_footer_js.py
  ```

На каждой странице должны быть элементы `<div id="site-header"></div>` и `<div id="site-footer"></div>` и подключён скрипт (путь с учётом глубины или от корня):
`<script src="/assets/js/header-footer.js"></script>`.

**Если добавили новые HTML-страницы вручную** и вставили в них старую шапку/подвал, замените их на placeholders и подключите скрипт. Либо запустите (осторожно, перезапишет подходящие файлы):
```bash
python3 scripts/replace_header_footer.py --dry-run   # посмотреть, какие файлы затронет
python3 scripts/replace_header_footer.py            # выполнить замену
```

## Описания товаров с bikzg.ru

Скрипт `fetch_descriptions_bikzg.py` подставляет в каталог описания товаров, взятые с сайта bikzg.ru (блок «Описание» со страницы товара). Результат попадает в поле `description` в `#products-data` и отображается в модалке «Подробнее».

### Кэш

Уже скачанные страницы сохраняются в файл **`.bikzg_fetch_cache.json`** в корне проекта (файл в .gitignore). При повторном запуске эти URL не запрашиваются снова. При выходе по **Ctrl+C** (или SIGTERM) кэш сохраняется перед завершением.

- **Отключить кэш:** флаг `--no-cache` (всё качать заново и не сохранять в кэш).

### Запуск из терминала

```bash
# Только тест (ничего не менять):
python3 scripts/fetch_descriptions_bikzg.py --dry-run --limit 5

# Обновить все страницы каталога (по умолчанию 4 потока, таймаут 60 с):
python3 scripts/fetch_descriptions_bikzg.py

# С большим числом потоков и логом каждого запроса:
python3 scripts/fetch_descriptions_bikzg.py --workers 8 --log-fetch

# Увеличить таймаут при нестабильной сети:
python3 scripts/fetch_descriptions_bikzg.py --workers 4 --timeout 90
```

### GUI

```bash
python3 scripts/fetch_descriptions_bikzg_gui.py
```

Открывается окно: выбор корня проекта, потоков, таймаута, файла лога. Полный лог пишется в файл; в интерфейсе — прогресс и итог (удачно / ошибки / без описаний). Кнопка «Стоп» завершает процесс с сохранением кэша.

### Вариант: сохранить описания в JSON, потом подставить

Удобно, если сеть нестабильна или нужно подставить описания без доступа к bikzg:

```bash
# Скачать описания и сохранить в файл:
python3 scripts/fetch_descriptions_bikzg.py --dry-run --save-json descriptions.json --limit 100

# Подставить сохранённые описания в HTML (сети не нужен):
python3 scripts/fetch_descriptions_bikzg.py --from-json descriptions.json
```

### Опции командной строки

| Опция | Описание |
|-------|----------|
| `--dry-run` | Не менять HTML, только скачать и вывести |
| `--limit N` | Обработать не более N страниц |
| `--only SUBSTR` | Обрабатывать только пути, содержащие подстроку SUBSTR |
| `--workers N` | Параллельных потоков (по умолчанию 4; при таймаутах лучше 2–4) |
| `--timeout N` | Таймаут одного запроса в секундах (по умолчанию 60) |
| `--no-cache` | Не использовать кэш (скачать всё заново и не сохранять) |
| `--log-fetch` | Выводить лог каждого запроса (кэш / сеть / повтор / OK) |
| `--save-json FILE` | Сохранить полученные описания в JSON |
| `--from-json FILE` | Взять описания из JSON, не качать с bikzg |
| `path` | Корень проекта (по умолчанию текущая директория) |

### Формат JSON для --from-json

Файл должен быть объектом «название товара» → «текст описания»:

```json
{
  "Автоматика деаэратора ДА-15/8": "Текст описания...",
  "Экономайзер ЭБ-1-300": "Другое описание..."
}
```

Сопоставление с товарами на сайте идёт по полю `title` в `products-data`.

### Зависимости

Установка `beautifulsoup4` (`pip install beautifulsoup4`) улучшает разбор HTML с bikzg; без неё используется парсинг по регулярным выражениям.
