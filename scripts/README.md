# Скрипты

## Шапка и подвал (один экземпляр на весь сайт)

Шапка и подвал хранятся в **одних файлах** и подставляются на все страницы через JS.

- **Фрагменты (редактировать здесь — меняется на всех страницах):**
  - `assets/html/header.html` — навигация (логотип, меню, кнопка «Связаться»)
  - `assets/html/footer.html` — подвал (логотип, ссылки, контакты, копирайт)
- **Подстановка:** скрипт `assets/js/header-footer.js` вставляет шапку и подвал сразу (HTML встроен в скрипт, без запросов к серверу).
- **После изменения** `header.html` или `footer.html` нужно пересобрать скрипт:
  ```bash
  python3 scripts/build_header_footer_js.py
  ```

На каждой странице должны быть элементы `<div id="site-header"></div>` и `<div id="site-footer"></div>` и подключён скрипт:
`<script src="/assets/js/header-footer.js"></script>` (со страниц в подпапках — с ведущим слэшем).

**Если добавили новые HTML-страницы вручную** и вставили в них старую шапку/подвал, замените их на placeholders и подключите скрипт. Либо запустите (осторожно, перезапишет все подходящие файлы):
```bash
python3 scripts/replace_header_footer.py --dry-run   # посмотреть, какие файлы затронет
python3 scripts/replace_header_footer.py            # выполнить замену
```

Шапка и подвал встроены в скрипт, поэтому подставляются сразу при загрузке страницы (в том числе при открытии с диска по `file://`).

## Описания товаров с bikzg.ru

Скрипт `fetch_descriptions_bikzg.py` подставляет в каталог описания товаров, взятые с сайта bikzg.ru (блок «Описание» со страницы товара). Результат попадает в поле `description` в `#products-data` и отображается в модалке «Подробнее».

### Вариант 1: Скачать с bikzg и сразу обновить HTML

```bash
# Только тест (ничего не менять):
python3 scripts/fetch_descriptions_bikzg.py --dry-run --limit 5

# Обновить первые 10 страниц каталога:
python3 scripts/fetch_descriptions_bikzg.py --limit 10

# Обновить все страницы каталога (долго, много запросов к bikzg.ru):
python3 scripts/fetch_descriptions_bikzg.py
```

Между запросами делается пауза ~0.8 с. При ошибках сети (SSL, таймаут) скрипт повторяет запрос до 2 раз.

### Вариант 2: Сначала сохранить описания в JSON, потом подставить

Удобно, если сеть нестабильна или нужно запускать обновление без доступа к bikzg:

```bash
# Скачать описания и сохранить в файл (можно с --limit):
python3 scripts/fetch_descriptions_bikzg.py --dry-run --save-json descriptions.json --limit 20

# Подставить сохранённые описания в HTML (сети не нужен):
python3 scripts/fetch_descriptions_bikzg.py --from-json descriptions.json
```

### Дополнительные опции

- `--only SUBSTR` — обрабатывать только те пути каталога, в которых встречается подстрока (например `avtomatika-deaeratora-da-15-8`).
- `--limit N` — обработать не более N страниц.
- `--workers N` — число параллельных потоков при загрузке с bikzg (по умолчанию 8). Ускоряет работу в несколько раз.
- Установка `beautifulsoup4` (`pip install beautifulsoup4`) улучшает разбор HTML с bikzg; без неё используется парсинг по регулярным выражениям.

### Формат JSON для --from-json

Файл должен быть объектом «название товара» → «текст описания»:

```json
{
  "Автоматика деаэратора ДА-15/8": "Текст описания...",
  "Экономайзер ЭБ-1-300": "Другое описание..."
}
```

Сопоставление с товарами на сайте идёт по полю `title` в `products-data`.
